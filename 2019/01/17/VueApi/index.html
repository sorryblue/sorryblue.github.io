判断

v-if

  • v-if=”数据|判断”,只要条件成立就显示if中的元素,根据表达式的值的真假条件渲染元素

  • 当它们处于同一节点, v-for 的优先级比 v-if 更高,这意味着 v-if 将分别重复运行于每个 v-for 循环中
    ###v-else

  • 限制: 前一个兄弟元素必须有 v-if 或v-else-if

  • 用法: 为 v-if 或者 v-else-if 添加’else块’

v-else-if

  • 限制: 前一兄弟元素必须有v-if 或者 v-else-if

  • 用法: 表示v-if的”else if 块”,可以链式调用

v-show

  • 根据表达式的真假,切换元素的display Css属性,当条件发生变化时该指令触发过度效果

  • 性能要比v-if略好

v-html

  • 预期: string

  • 更新元素的innerHTML.注意:内容按普通HTML插入-不会作为Vue模板进行编译,如果试图使用v-html组合模板,可以重新考虑是否通过使用组件来替代

  • 在网站上动态渲染任意 HTML 是非常危险的，因为容易导致 XSS 攻击。只在可信内容上使用v-html，永不用在用户提交的内容上。

v-on

  • 缩写: @

  • 预期: Function | Inline Statement | Object

  • 参数: event

  • 修饰符:

    • .stop - 调用event.stopPropagetion()
    • .prevent - 调用event.preventDefault()
    • .capture - 添加事件侦听器时使用captrue模式 -> 在进行冒泡的时候,先从上向下触发添加了capture的事件,然后是触发的事件,然后进行冒泡
    • .self - 只当事件是从侦听器绑定的元素本身触发时才触发回调,如果不是直接触发此事件,如冒泡,就会跳过这个事件
    • .{keyCode | keyAlias} - 只当事件是从特定键触发时才会触发回调,keyCode可以直接写键别名如.enter
    • .native - 监听组件根元素的原生事件 ->vue自己定义的标签 <router-link> 还有 组件<my-vue-template/> 没有点击事件,H5不论什么白标签天生就有点击事件.加上.native就变身H5标签
    • .once - 只触发一次回调
    • .left - 只当点击鼠标左键时触发
    • .right - 只当点击鼠标右键时触发
    • .middle - 只当点击鼠标中键时触发
    • .passive - 以{passive: true}模式添加侦听器

  • 事件函数写在methods中,methods在根目录下,值为一个对象

  • 可以在行内事件函数中传入参数

v-model

  • 预期: 随表单控件类型不同而不同
  • 限制:
    • <input>
    • <select>
    • <textarea>
    • components
  • 修饰符:
    • .lazy- 取代input监听change事件
    • .number- 输入字符串转为有效数字
    • .trim - 输入首位空格过滤
  • v-model会忽略所有表单元素的value,checked,selected特性的初始值而总是将 Vue 实例的数据作为数据来源。你应该通过 JavaScript 在组件的data` 选项中声明初始值。
  • 对于需要使用输入法(如中文、日文、韩文等)的语言,你会发现 v-model 不会在输入法组合文字过程中得到更新。如果你也想处理这个过程，请使用 input事件.
  • ==height== h~text~ 划掉 typora特殊指令
  • model实例
  • 在select中,直接绑上v-model,如果option中有value,选择的结果优先value值，如果没有value值，走option的内容。
