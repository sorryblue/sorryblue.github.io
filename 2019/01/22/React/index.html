jsx

jsx => javascript + xml 一种js和自定义写法的一种混合模式

jsx语法主要是为了更好的读写模板或者组件,jsx语法浏览器是不是别的,通过babel的方式把jsx转成浏览器能够识别的结构

在react中，写<div></div>,其实就等同于写了React.createElement('div',children,'click','active')

jsx规则

1. 遇到样式中的`class`,直接变成`className`
2. 标签必须闭合,特别需要注意单标签
3. `react` 中只需要一个{ },vue需要两个{ }
4. { }中能够放任何得js代码
5. 默认展开数组
6. 注释
7. 可以申明函数,但是不能直接调用
8. `value={a}`赋值某个元素的属性
9. 设置`style={{width:'200px',height:'200px'}}`
10. 表单元素设置默认值的时候,就会报错,因为他会认为input为一个动态组件,是动态就会操作数据(value就一定会变化),一定会变化,就一定要有事件(onChange),所以才会报错,有两种解决办法:
   1. 就给表单元素加上事件（onChange）  (受控组件)官方更加推荐使用
    <input value="1" onChange={ } />
    2. 定义默认值的时候就使用defaultValue（非受控组件）
    <input defaultValue="1" />

写方法的时候,如果不对函数做处理,事件函数的this默认指向undefined

解决方案:

  1. 使用create-app click=()=>{ }

  4. click(){ } <button onClick={(e)=>this.click(e)>

    state状态

    操作状态中的数据,就能操作视图

操作数据不能直接使用this.state.xx = ooo
只能通过this.setState()去修改你的状态
this.setState(对象(就是state的数据),()=>{当操作完数据之后的回调回调函数})

this.setState(函数(prev)=>{return prev+1},()=>{当操作完数据之后的回调函数})

this.state = {} 状态初始化 初始化必须放在constructor下

只要更新state就会更新视图

从父级传进来的为props

父级传子级，把数据挂在子组件的属性上子组件接收父组件的数据通过this.props

表单如果要设置默认值

子传父

  1. 父级需要定义一个修改数据的方法
  2. 把修改数据的方法传给子组件
  3. 当子组件需要修改父级数据的时候，调用父级传过来的修改方法
  • 方法:(把父级的数据变成子级，子级的数据不跟父级的数据有瓜葛了)
    1. 把父级的数据变成子级的
